1. Theory Section
Core concepts · Paths · File operations

Absolute vs relative paths

This is the single most important navigation concept. Get it wrong and every command fails.

Absolute path
Starts with / — always the same no matter where you are. Example: /etc/passwd
Relative path
Starts from your current location. Example: ../home/user goes up one level then into home/user

Special path shortcuts

~
Your home directory — expands to /home/username or /root for root
.
Current directory — used in commands like ./script.sh
..
Parent directory — cd .. goes up one level
-
Previous directory — cd - jumps back to where you just were
/
Root of the entire filesystem — top of the tree

Navigation commands

pwd
Print working directory — "where am I right now?"
cd <path>
Change directory — your primary movement tool
ls
List directory contents. Use -la always (hidden files + permissions)
tree
Visual directory tree. Great for mapping structure at a glance

File operation commands

touch <file>
Create empty file or update timestamps on existing file
mkdir -p <dir>
Create directory. -p creates parent directories too
cp -r <src> <dst>
Copy file or directory. -r for recursive (directories)
mv <src> <dst>
Move or rename. Same command for both operations
rm -rf <target>
Delete file or directory. -r recursive, -f force — no undo, use carefully
ln -s <src> <link>
Create a symbolic link (shortcut). Critical to understand for security

Reading file contents

cat <file>
Print entire file to screen. Fast but overwhelming for large files
less <file>
Page through file interactively. Press q to quit, /term to search
head -n 20
Show first 20 lines. Default is 10
tail -n 20
Show last 20 lines. -f flag follows file live (perfect for logs)
file <target>
Identify file type regardless of extension — critical in forensics
xxd <file>
Hex dump — see raw bytes of any file. Used in malware analysis

Searching for files

find <path> -name
Search by filename. Supports wildcards: find / -name "*.conf"
find -type f/d
f = files only, d = directories only
find -mtime -1
Files modified in the last 1 day — critical for incident response
find -size +10M
Files larger than 10MB — find data exfiltration staging
locate <name>
Faster than find — uses a pre-built index. Run updatedb to refresh
which <command>
Find where a binary lives: which python3 → /usr/bin/python3

2. Practical Section
Hands-on exercises · Build muscle memory

Exercise 1 — navigation drills

pwd # confirm where you are cd / # go to root ls -la # list with hidden + permissions cd /etc # jump to /etc cd ../var/log # relative: up one, into var/log cd ~ # back to home instantly cd - # toggle back to /var/log echo $PWD # another way to see current path

Exercise 2 — build a practice directory tree

mkdir -p ~/security-lab/{notes,scripts,evidence,tools} ls ~/security-lab # verify structure tree ~/security-lab # visual tree (install: apt install tree) mkdir -p ~/security-lab/evidence/day3 touch ~/security-lab/notes/day3-commands.txt echo "Day 3 notes" > ~/security-lab/notes/day3-commands.txt cat ~/security-lab/notes/day3-commands.txt

Exercise 3 — copy, move, rename

cp /etc/hostname ~/security-lab/evidence/hostname_copy cp -r /etc/ssh ~/security-lab/evidence/ssh_backup # copy directory mv ~/security-lab/evidence/hostname_copy ~/security-lab/evidence/hostname_renamed ls -la ~/security-lab/evidence/

Exercise 4 — reading files like a pro

cat /etc/passwd # view whole file less /etc/passwd # paginate (q to quit) head -5 /etc/passwd # first 5 lines tail -5 /etc/passwd # last 5 lines tail -f /var/log/syslog # live log follow (Ctrl+C to stop) wc -l /etc/passwd # count lines in file

Exercise 5 — find files like an attacker

find /home -name "*.txt" 2>/dev/null find / -name "id_rsa" 2>/dev/null # hunt for SSH private keys find / -name "*.conf" -type f 2>/dev/null | head -20 find /tmp -type f -newer /tmp 2>/dev/null find / -mtime -1 -type f 2>/dev/null | head -20 # modified last 24h find / -size +5M -type f 2>/dev/null | head -10 # large files

Exercise 6 — file type identification (forensics basics)

file /bin/bash # shows: ELF 64-bit executable file /etc/passwd # shows: ASCII text file /var/log/syslog # shows: ASCII text (with timestamps) file ~/security-lab/notes/day3-commands.txt xxd /bin/bash | head -3 # see ELF magic bytes: 7f 45 4c 46

Exercise 7 — symlinks (understand for privilege escalation)

ln -s /etc/hostname ~/security-lab/tools/hostname_link ls -la ~/security-lab/tools/ # see the -> arrow pointing to target cat ~/security-lab/tools/hostname_link # reads through the link readlink -f ~/security-lab/tools/hostname_link # resolve full real path
3. Real-World Scenario
Attack & defense story

An attacker gains a low-privilege shell on a web server via a file upload vulnerability. They now use navigation and file operations to escalate and persist — everything they do today is what you practiced.

Step 1 — orient and enumerate

They run pwd, id, uname -a. They're www-data in /var/www/html. They run ls -la and find a .env file the developer left in the web root containing database credentials.

Step 2 — search for interesting files

They run find / -name "*.conf" 2>/dev/null and find a backup config at /opt/app/config.bak with SSH credentials in plaintext. They also run find /home -name ".bash_history" 2>/dev/null and read the developer's command history.

Step 3 — copy evidence and stage tools

They cp the config files to /tmp/.attacker/ (hidden directory via dot prefix), then use find / -mtime -7 -type f 2>/dev/null to see what files changed in the last 7 days — revealing recently deployed credentials.

Step 4 — cover tracks with mv and touch

They rename their dropped files to blend in, use touch -t to fake timestamps on modified files, and delete their working directory with rm -rf /tmp/.attacker/ before leaving.

Defender lesson

No plaintext credentials in accessible locations. File integrity monitoring would catch new files in /tmp. Monitoring find and cp commands in auditd logs would expose the recon. Restricting what www-data can read with proper permissions (Day 4) is the real fix.

4. Red Team Perspective
How attackers use navigation and file ops

Post-exploitation navigation checklist

The moment an attacker gets a shell, they follow a mental checklist using exactly the commands from today. Speed matters — defenders may detect and kill the session.

id && whoami && hostname # who am I and where uname -a # OS + kernel version for exploit selection cat /etc/os-release # distro details ls -la /home # other users on the system cat ~/.bash_history # what has this user done recently env # environment variables — often has secrets find / -perm -4000 2>/dev/null # SUID binaries for privesc find / -writable -type f 2>/dev/null | grep -v proc # writable files

Data exfiltration file operations

find / -name "*.sql" 2>/dev/null # database dumps find / -name "id_rsa*" 2>/dev/null # SSH private keys find / -name "*.pem" 2>/dev/null # SSL/TLS private keys find / -name "*.kdbx" 2>/dev/null # KeePass password databases cp /etc/shadow /tmp/.s && cat /tmp/.s # copy and read shadow file tar czf /tmp/.loot.tgz /home/ 2>/dev/null # archive home dirs for exfil

Hiding in the filesystem

  • Use dot-prefix names: mkdir /tmp/.hidden — invisible to plain ls
  • Use touch -t 202201010000 to backdate file timestamps
  • Write to /dev/shm — lives in RAM, no disk writes, harder to detect
  • Use mv to rename payloads to look like system files
  • Overwrite rather than create: less likely to trigger new-file alerts
5. Blue Team Perspective
Detect suspicious file operations

Audit file access with auditd

Linux's audit daemon can log file reads, writes, and executions. This is how defenders catch attackers doing the same find and cat commands you practiced today.

auditctl -w /etc/passwd -p r -k passwd_read # alert on reads of passwd auditctl -w /etc/shadow -p r -k shadow_read # alert on reads of shadow auditctl -w /tmp -p x -k tmp_exec # alert on execution in /tmp ausearch -k shadow_read # search audit log for that key

Detect recent file changes (incident response)

find / -mtime -1 -type f 2>/dev/null # files changed last 24h find /tmp /dev/shm -type f 2>/dev/null # unexpected files in temp areas find / -name ".*" -type f 2>/dev/null | grep -v "^/proc" # all hidden files ls -lat /etc/ | head -20 # most recently modified in /etc stat /etc/passwd # check if timestamps were tampered

Detect hidden directories

find /tmp /var/tmp /dev/shm -name ".*" 2>/dev/null find / -maxdepth 4 -name ".*" -type d 2>/dev/null | grep -v "^/home\|^/root\|^/proc"

Protect against destructive rm -rf

  • Use immutable flags on critical files: chattr +i /etc/passwd
  • Regular backups — rsync, tar, or dedicated backup tools
  • Restrict shell access — attackers can't run rm if they can't get a shell
  • Monitor with auditd for mass deletion events — sudden spike in unlink syscalls

10. Pro Tips & Mindset
Habits that separate beginners from professionals

Always use ls -la, never just ls. The -a flag reveals hidden files (dot-prefixed). Every single professional does this automatically. Make it a reflex from Day 3.

Use Tab completion constantly. Start typing a path, press Tab — the shell autocompletes. Double-Tab shows all options. This eliminates typos and speeds you up dramatically.

Use Ctrl+R to search your command history. Start typing any part of a previous command and it appears. Far faster than pressing the up arrow 30 times.

Pipe commands together. find / -name "*.log" 2>/dev/null | wc -l counts all log files. The pipe | passes output of one command into the next. This is one of Linux's most powerful features.

Common mistake: running rm -rf without double-checking the path. There is no recycle bin in Linux. One wrong path and critical files are gone permanently. Always verify with ls first.

Common mistake: confusing mv and cp. mv removes the source. cp keeps it. When in doubt during incident response, always cp — preserve the original.

Pro mindset: think in pipelines, not individual commands. Real work looks like: find /var/log -mtime -1 -type f | xargs grep -l "Failed password" 2>/dev/null — find recent log files then search each for failed passwords. One line, massive power.
